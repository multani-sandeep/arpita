### ​​**Glossary of Terms**

* **Algorithm**: A sequence of computational instructions that enables a computer to perform a task such as object recognition or predictive analytics  
* **Algorithm hallucination** \- Unlike humans, AI doesn't know how to say, "I don't know." In other words, it might make up things. This is a serious limitation of AI and can occur primarily when the algorithm trying to overfit on a small or biased data set  
* **Artificial Intelligence (AI)**: A field of computer science that studies how to create machines that can replicate cognitive tasks such as problem-solving, reasoning, perception, and language understanding \- e.g. Youtube recommendation \- automatically using a computer program   
* **Artificial General Intelligence (AGI):** Artificial intelligence that is capable of solving a wide range of problems, much as humans can, rather than excelling at one specific task  
* **Autonomous agents with tools** \- these are AI powered software entities that run on their own rather than waiting for you to prompt them \- you give it a high level mission and let the agent to work in the background \- here prompt engineering is even more important.   
* **ChatGPT:** A conversational AI model based on GPT, which can be used to create chatbots or interact with users in natural language  
* **ChatGPT** \- ChatGPT, started as LLM. is a generative artificial intelligence chatbot developed by OpenAI. It uses generative pre-trained transformers (GPTs), such as GPT-4o or o3, to generate text, speech, and images in response to user prompts. Talks like a human person.  \- based on the earlier model GPT-3. Geared towards chatting and conversation. It’s a more scaled down, more-specialised version of GPT-3.  
* **Classification:** The categorization of various forms of AI based on features such as the type of data they process, the tasks they can perform, and the specific techniques or algorithms they employ  
* **Computing Power:** The amount of processing power available to a computer system, which can impact the speed and accuracy of AI algorithms  
* **Context window \-** A context window is the amount of tokens you can have in the input and output when interacting with a large language model. So a very large context window will allow for a very long chat without losing too many details. Gemini has a long context window.   
* **Deep Learning**: A type of machine learning that uses multilayered neural networks to extract high-level patterns from data  
* **Diffusion Model:** Model that starts with a noisy sequence and then slowly removes noise until it has a clean image.  A model that is used to learn the latent structure of a data set by modeling the way data points diffuse through a latent space; this technique can be applied to various tasks such as image generation, denoising, and super-resolution  
* **Fine-Tuning:** The process of making small changes to an AI algorithm to improve its performance  
* **Few Shot Learning** is where we teach a model to do something by showing it a few structured examples as opposed to giving it an explicit instruction. When we just give an instruction without examples, that's often referred to as **Zero Shot Learning.** One of the advantages of Few Shot Learning is that you can have fine tuned control over the output of the model.  
* **Generative** \- creating new content (audio, code, images, text, video)  
* **Gen AI \-** Generative Artificial Intelligence \- AI that generates new original content  
* **GPT**: The Generative Pretrained Transformer model, which is a deep learning language model used for natural language processing (NLP) tasks  
* **GPT** \- Generative Pre-training Transformer \- uses the Transformer architecture   
* **GPT-3** \- more general purpose and powerful AI model   
* **GPUs** or computing power constraints \-  GPU stands for graphic processing units, and they're often used for parallel processing in AI applications  
* **Language Models  \-** Can take a prompt and create a completion by constantly predicting the next token. This is an incredibly difficult task, and in order to achieve this, these models are trained on large amounts of data, including almost all of the crawlable web, a lot of literature, and they're often fine tuned to become more conversational or to be better at a particular set of skills.  
* **Large Language Model (LLM)**: A subset of AI that allows machines to understand and process large quantities of text.   
* **LLM \- Large Language Models** \- type of Generative AI that can communicate using normal human language \- LLM is an artificial neural network   
  * ChatGPT \- powered by GPT-4 and GPT-4o   [chatgpt.com](http://chatgpt.com)

* **Machine Learning:** A subset of AI that enables computers to discover patterns in data in order to make predictions or decisions on their own  
* **Mixture of Experts \-** LLM architecture, Gemini from Google uses this architecture, along with the Transformer architecture.   
* **Modality \-** modality refers to the types of data an AI system can understand and process ie, various input data  data formats:   
  * **Visual**: Images, videos;    
  * **Textual**: Text documents, code;   
  * **Audio**: Speech, music;   
  * **Sensor data**: Temperature readings, pressure gauges (data from Internet of Things)  
* **Neural Networks** \- a bunch of artificial nodes connected to each other \- any content like text or images can be represented as numbers and when you feed this into the the neural network it guesses the next word / content as a set of numbers \- it is Guess the next word machine \- now when you get the output and feed it back to the Neural Network it continues to generate large language outputs   
* **OpenAI \-** is an American artificial intelligence (AI) organization that developed GPT-3, ChatGPT etc. OpenAI APIs \- companies can use the APIs to streamline processes and lead to cost-savings.   
* **Prompt engineering** \- In order to use Generative AI effectively, you need to get good at Prompt Engineering or Prompt Design.  This skill is needed both as a user and as a product developer.  Prompt engineering refers to constructing inputs that help us get the most out of generative AI and language models in particular  
* **Responsible AI**: Ensuring that AI applications are designed in a way that minimizes bias, preserves privacy, and meets regulatory requirements  
* **RLHF** \- **Reinforcement Learning Human Feedback** \- What really makes ChatGPT stands out is a process called RLHF, reinforcement learning human feedback. This is a part of the model's training where people give feedback on the model's responses, making it extremely powerful when it comes to following instructions.  
* **Synthetic Data:** Artificially generated data that can be substituted for real-time data to train AI algorithms  
* **Token \-** a unit that can easily be understood by a language model. E.g. Word \- Everyday, Tokens \- Every, day. A small unit that can be used by a large language model.  
* **Tokenisation** \- The mechanism by which a model splits its inputs into tokens.  Different models split inputs into tokens in different ways.   
* **Token constraints** \- A token refers to a unit that the generative AI system is able to process at one time. It puts a limit on the amount of text that the system is able to accommodate, whether as input to the system or output out of the system. The amount of input is critical for success. The more you can bring nuanced and unique input into your prompt, the better and more accurate your solution could be. Token constraints can limit the types of data that can be used for training or the scope of problems that the system can address  
* **Transformers**: A type of neural network that uses self-attention mechanisms to process large sequences of data \- They use a technique of "self-attention" to identify the most essential parts of an input.  
* **Training Data**: A set of data that is used to help “train” an AI algorithm so that it can learn to improve its performance  
* **Workflow Automation:** The use of software technologies to automate recurring tasks within a business process  
* **Zero shot learning** \- Zero-shot learning is described as prompting a language model with an instruction without really giving it examples of the task you want it to perform.  This is slightly different to **Few Shot Learning**, where we teach a model to do something by showing it a few structured examples as opposed to giving it an explicit instruction.
